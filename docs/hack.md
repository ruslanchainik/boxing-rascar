# Boxing Action Recognition Challenge / Rascar Box

## Overview

В соревновании нужно разработать алгоритм для автоматического анализа боксерских поединков по видео.

На вход подаются видеозаписи раундов, снятые одной статичной камерой из-за ринга. Нужно найти все моменты ударов и определить, какой спортсмен нанес каждый удар: `red` или `blue`.

Сабмит представляет собой таблицу событий:

- одна строка = один предсказанный удар;
- для каждого события нужно указать кадр удара;
- нужно указать атакующего спортсмена;
- также нужно заполнить дополнительные атрибуты удара.

Ключевая часть задачи:

- корректно найти момент удара;
- определить бойца, который нанес удар.

Дополнительные характеристики удара улучшают итоговое качество решения.

## Task Description

Датасет содержит видеозаписи любительских боксерских турниров и тренировок.

Видео нарезаны по раундам. Все записи сняты одной статичной камерой, расположенной за рингом и покрывающей всю площадку.

В кадре обычно видны:

- оба спортсмена;
- рефери;
- часть окружения;
- иногда зрители или другие визуальные помехи.

Спортсмены обозначаются как `red` и `blue`. Это не имена бойцов, а стороны в рамках конкретного видео.

## Экипировка и цвет спортсменов

Цвет спортсмена определяется по экипировке:

- шортам;
- майке;
- перчаткам;
- шлему, если он присутствует.

На турнирных видео экипировка обычно хорошо соответствует цвету спортсмена:

- красный спортсмен выступает в красной форме;
- синий спортсмен выступает в синей форме.

На тренировочных видео ситуация сложнее:

- часть экипировки может быть нейтрального цвета, например черного;
- нельзя полагаться только на цвет отдельных элементов формы;
- важно учитывать устойчивый трек спортсмена, положение в кадре и поведение на протяжении раунда.

## Что Нужно Предсказать

Модель должна проанализировать test-видео и вернуть список найденных ударов.

Для каждого удара нужно определить:

- кадр, на котором произошел удар;
- спортсмена, который нанес удар;
- тип удара;
- бьющую руку;
- цель удара;
- результат удара;
- был ли удар чистым или в нем присутствовал шум/заслонение.

Кадры нумеруются с нуля внутри каждого mp4-файла, как в `train/punches.csv`.

## Почему Задача Сложная

Задача требует анализа движения во времени, а не только классификации отдельных кадров.

Модели нужно отличать реальный момент удара от:

- замаха;
- подготовки;
- перемещения;
- защитного движения;
- клинча.

Основные сложности:

- быстрые движения рук и корпуса;
- перекрытия спортсменов друг другом;
- присутствие рефери в кадре;
- различия между турнирными и тренировочными видео;
- неоднозначные ситуации с попаданием, блоком и промахом;
- возможные заслонения и визуальный шум;
- необходимость не генерировать лишние удары на спокойных фрагментах.

Рекомендуется использовать временной контекст:

- последовательности кадров;
- keypoints;
- треки спортсменов;
- движение рук;
- дистанцию между бойцами;
- динамику корпуса.

## Evaluation

Метрика оценивается macro-average по видео.

Для каждого test-клипа предсказанные удары сопоставляются с ground truth по времени:

- используется алгоритм Венгера;
- окно сопоставления: `±1.0` секунды;
- fps считается как 30;
- время события: `t = frame / 30`.

В score входят:

- момент удара: 50%;
- боец: 20%;
- тип удара: 10%;
- результат: 8%;
- рука: 6%;
- цель: 6%;
- штраф за лишние предсказания FP.

В ground truth учитываются только удары с `clear=true`.

Строки сабмита с `clear=false`:

- не матчатся;
- не считаются FP.

Итоговый score в диапазоне `[0, 1]`, выше -- лучше.

## Формула Скора За Одно Видео

```text
final_score = 0.50 * score_time
            + 0.20 * score_fighter
            + 0.10 * score_punch_type
            + 0.08 * score_effectiveness
            + 0.06 * score_hand
            + 0.06 * score_target
            - fp_penalty
```

Результат зажимается в `[0, 1]`.

Лидерборд -- macro-average по всем test-видео, у которых есть GT с `clear=true`.

## Шаг 1: Сопоставление

Предсказания сопоставляются с GT алгоритмом Венгера по временному расстоянию.

Окно:

```text
|t_gt - t_pred| <= 1.0
t = frame / 30
```

При равном времени у кандидатов приоритет у пары с тем же `fighter`.

Типы исходов:

- TP: GT-удар с предсказанием в окне, one-to-one;
- FN: GT-удар без пары;
- FP: предсказание без GT в окне.

## Шаг 2: Ошибка По Времени

Для каждого TP:

```text
time_error = min(1.0, |t_gt - t_pred| / 0.5)
```

Примеры:

| Ошибка времени | `time_error` |
|---:|---:|
| 0.00 с | 0.0 |
| 0.25 с | 0.5 |
| >= 0.50 с | 1.0 |

Если `time_error = 1.0`, ошибки атрибутов для этой пары считаются максимальными.

Для `punch_type` и `effectiveness` максимальные ошибки считаются с весом класса GT.

## Шаг 3: Ошибки По Атрибутам

Для TP с `time_error < 1.0`:

- ошибка 0, если значение совпало с GT;
- ошибка 1, если значение не совпало;
- для `punch_type` и `effectiveness` при промахе используется вес класса GT.

Для FN:

- максимальный штраф по всем компонентам;
- для взвешенных полей используется вес класса GT.

Сравниваются:

- `fighter`;
- `punch_type`;
- `hand`;
- `target`;
- `effectiveness`.

## Шаг 4: Компонентные Оценки

Для `time`, `fighter`, `hand`, `target`:

```text
score_X = 1 - (сумма ошибок по X) / N_gt
```

Для `punch_type` и `effectiveness` используется balanced accuracy с весами обратной частоты по классам GT:

```text
w_c = N_total / (N_classes * N_c)
```

Знаменатель -- сумма `w_c` по всем GT-ударам видео.

Редкие классы, например `uppercut` и `blocked`, дают больший вклад при угадывании.

Внутри метрики `effectiveness` приводится к:

| Сабмит | Внутри метрики |
|---|---|
| `landed` | `hit` |
| `blocked` | `block` |
| `miss` | `miss` |

## Шаг 5: Штраф За FP

```text
fp_penalty = N_fp / (N_gt + N_fp)
```

Примеры:

| FP / GT | Штраф |
|---:|---:|
| 0 | 0.00 |
| 0.5x | 0.33 |
| 1x | 0.50 |
| 4x, около лимита 5x | 0.80 |

Вывод:

- лучше недопредсказать, чем сильно перепредсказать;
- `clear=false` безопасен для заполнения лишних строк;
- главный фокус решения: точные пики `frame` и правильный `fighter`.

## Практические Следствия Для Решения

Самые важные части:

1. Найти момент удара.
2. Определить атакующего бойца.
3. Не нагенерировать много FP.

Из-за весов метрики хороший базовый pipeline должен быть temporal:

- pose/keypoints;
- треки двух бойцов;
- последовательности движения кистей, локтей, корпуса;
- расстояние кисти до головы/корпуса соперника;
- устойчивое назначение `red` / `blue`.

Цвет полезен для роли бойца, но опасен как единственный сигнал:

- синий ринг может давать ложные blue-пиксели;
- нужна маска по телу/перчаткам через keypoints;
- желательно сглаживать роль на всем треке.
